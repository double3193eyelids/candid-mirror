# 魏征 Tips 知识库

> 魏征内置的小型知识库,不依赖联网搜索。按画像×场景匹配,每次挑1条最相关的放在报告里。
> 随版本迭代可补充,当前20条。

## 通用型(适用于所有用户)

| ID | Tips内容 | 匹配场景 |
|----|----------|----------|
| G01 | 策略完美主义是Agent用户最常见的效率陷阱——想得太多,做得太少 | 改轮次高但工具调用少 |
| G02 | 如果你发现自己反复新建会话做同一件事,说明你需要一个模板而不是更多耐心 | 同项目频繁新建会话 |
| G03 | 最好的Agent用户不是用得最多的,而是每次用完都总结一条笔记的 | 通用,无特定触发 |
| G04 | 会话超过30轮还没结果,通常不是AI的问题,是需求没想清楚 | 深度迭代型会话 |
| G05 | 周末还在用Agent,不是勤奋,是没有边界。设一个"断电时间"试试 | 周末/节假日高频 |
| G06 | 你的Agent时间分布就是你的注意力分布——看看它,你就知道自己在逃避什么 | 通用,首次运行 |
| G07 | 工具调用种类少不一定是问题,可能你找到了最高效的路径 | 工具种类少 |
| G08 | 数据不够的时候不做判断,这是魏征的底线。你也一样——别用3天数据做月度规划 | 数据不足降级 |

## 赚钱型(ROI导向)

| ID | Tips内容 | 匹配场景 |
|----|----------|----------|
| M01 | 你的Agent时间花在哪,钱就从哪来。先看时间分布,再算投入产出 | purpose=赚钱 |
| M02 | 执行型任务改轮次超过15轮,ROI就在急速下降——写清楚需求比多改几轮值 | 执行型高改轮次+赚钱 |
| M03 | 赚钱用户的典型陷阱:花80%时间在"优化流程",只有20%在"交付赚钱" | 探索型占比高+赚钱 |
| M04 | 如果你最常用的工具是"搜索"而不是"写入文件",你还在研究阶段 | 搜索>写入 |

## 创作型(内容创作者)

| ID | Tips内容 | 匹配场景 |
|----|----------|----------|
| C01 | 3个Agent分工通常比1个主力产出更高——一个出创意、一个写正文、一个做排版 | 单Agent占比>70%+创作 |
| C02 | 创作型用户最常见的模式:先深度探索30轮,然后1轮写出成品。别把探索当浪费 | 探索型会话+创作工具 |
| C03 | 内容创作的效率瓶颈不在写作,在选题。让Agent帮你分析选题比你写10篇更有价值 | 通用创作场景 |
| C04 | 你的深夜创作效率可能比白天高——但只有数据能证明,不要靠感觉 | 深夜会话+创作 |

## 开发型(程序员)

| ID | Tips内容 | 匹配场景 |
|----|----------|----------|
| D01 | 干净session运行分析类Skill成本降低40%——上下文重复是token杀手 | 通用开发场景 |
| D02 | 调试类任务不要用长会话,每次bug开新会话,上下文更干净 | 高频短会话+开发 |
| D03 | 你的写入文件频率远高于读取文件，写得多看得少，大概率在边写边推翻重来——试试先读再写 | 写入>>读取 |
| D04 | exec调用集中在一个时段,说明你在批量部署——考虑自动化掉重复的部分 | exec集中+开发 |

## 匹配优先级

1. 先看预警/建议类型,匹配对应场景的tips
2. 再看用户画像(purpose),匹配对应类型
3. 没有精准匹配,从通用型(G01-G08)中选
4. 不重复上次展示的tips(可用yaml记录last_tips_id)
